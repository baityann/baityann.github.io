C胡言乱语（3）

发现一个问题，C++是面向对象的语言，那么有没有像java里面那样的CLASS呢？之前跟的项目里面，是namespace划分的。随便看了一下，发现是有class这个东西的。这样问题就变成了C++ Class和namespace的区别。
发现一点，命名空间可以在其它地方声明一下，然后往里面放东西，而类不可以再次定义一个。命名空间是个贴标签的东西，class是个小世界，别人访问有限制。
那么问题来了，什么时候用命名空间什么时候用Class呢？母鸡啊。stackocerflow上面找到那么一个

If you are writing code that you want to be associated with an object that you can define and use as a variable, write a class. If you are writing an API or library and you want to wrap up all of the functions and constants so that their names don’t clash with anything that the user might have written, use a namespace.

觉得现在的情况，C要调用C++的东西，C++用别人C++的库，因为别人的接口是C++格式的，所用C不能直接调用，要用C++做个接口出来给C用。 这样用namespace就够了。
迷的是下面两个方案，应该用哪个

  1. 用C的思想，做出每一步的接口，然后C调用
  2. 用C++ 的面向对象的思想处理C++部分，然后用一个接口，完成读取数据或者写入数据

还要考虑到要用到多线程，如果用一个类来处理，线程的函数怎么用。
如果当做对象来处理，好像有点大材小用了。

如果用namespace来处理，然后写一个综合的接口。不知所措。。。

如果这样的话，可以把一个东西放到全局变量里面来了。结构还要改。
